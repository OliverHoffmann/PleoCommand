Kommandozeilen Argumente

Argument (lang) Argument (kurz) Beschreibung
--detailed -d Gibt detaillierte (Debugging) Informationen aus.
--help -h -? Gibt diese Hilfe aus.
--version -V Gibt Versionsnummer und Copyright aus.
--gui -g Startet die graphische Benutzeroberfläche und zeigt Fehlermeldungen im Fehler-Dialog an (Standard, falls keinerlei Argumente angegeben wurden).
<PCA-Datei> Lädt eine zuvor über die GUI gespeicherte Pipe und führt diese aus, nachdem alle weiteren Argumente ausgewertet wurden.
Nach der Datei aufgeführte Argumente wie --configure wirken sich auf die geladene Pipe aus. Weitere PCA-Dateien überschreiben die vorhergehenden und führen keine zusätzliche Pipe aus.
--configure -c Ändert die Konfiguration der aktuell geladenen Pipe.
Syntax: <Name der Komponente>:<Bezeichner>:<Wert> oder <UID der Komponente>:<Bezeichner>:<Wert>.
Falls der Name der Komponente nicht eindeutig oder unbekannt ist, wird eine Fehlermeldung ausgegeben.
Falls die UID ungültig oder unbekannt ist, wird eine Fehlermeldung ausgegeben.
Falls der Bezeichner der Komponente unbekannt ist, wird eine Fehlermeldung ausgegeben.
Falls der Wert ungültig ist, wird eine Fehlermeldung ausgegeben.

Beispiele:
Eine Pipe in der GUI ausführen:
pleocommand MyPipe.pca --gui
Eine Pipe auf der Konsole mit Debugging Informationen ausführen:
pleocommand MyPipe.pca --detailed
Eine Pipe auf der Konsole ausführen, aber zuvor deren Konfiguration anpassen:
pleocommand MyPipe.pca --c=TcpIpInput:Port:1234 --c=TcpIpInput:"Read-Timeout (sec):60"
