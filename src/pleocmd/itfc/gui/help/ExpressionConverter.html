Expression Converter

Wertet Eingaben basierend auf einem mathematischen Ausdruck aus.
Alle Operatoren in Vorrangsreihenfolge:
|| Boolesches Oder: Ergibt 1, falls mindestes einer der beiden Werte 1
&& Boolesches Und: Ergibt 1, falls beide Werte 1
== != Boolescher Vergleich: Ergibt 1, falls beide Werte identisch / unterschiedlich (*1)
< <= > >= Boolesche Vergleichswerte: Ergeben 0 der 1 (*1)
+ - Addition, Subtraktion mit Fließkommazahlen
* / % Multiplikation, Division, Modulo mit Fließkommazahlen
! Negation: Ergibt 0 bei Werten ungleich 0, sonst 1
# Ergibt Inhalt eines Eingabekanals (0 bis 31), ergibt 0 bei nicht vorhandenem Kanal
^ Potenz Funktion
<Name> Ergibt Ergebnis einer Konstanten
<Name>() Ergibt Ergebnis nach Aufruf einer Funktion ohne Argument
<Name>(<Argument 1>) Ergibt Ergebnis nach Aufruf einer Funktion mit einem Argument
<Name>(<Argument 1>, <Argument 2>) Ergibt Ergebnis nach Aufruf einer Funktion mit zwei Argumenten
( ) Klammerung von Ausdrücken

Alle Konstanten:
e Die Konstante e
pi Die Konstante pi

Alle nullstelligen Funktionen:
rand Fließkomma - Zufallszahl zwischen 0 und 1

Alle einstelligen Funktionen:
abs Betrag
acos Arkuskosinus
acosh Inverser Kosinus Hyperbolikus
asin Arkussinus
asinh Areasinus Hyperbolikus
atan Arkustangens
atanh Inverser Tangens Hyperbolikus
cbrt Kubische Wurzel
ceil Nächste ganze Zahl größer oder gleich dem Parameter
cos Kosinus
cosh Kosinus Hyperbolikus
erf Fehlerfunktion: 2 / sqrt(pi) * Integral von 0 bis x über exp(-t * t) dt
erfc Komplementärfehler
exp e hoch Argument
floor Nächste ganze Zahl kleiner oder gleich dem Parameter
lgamma Logarithmus des Betrages der Gamma-Funktion (siehe tgamma)
log Natürlicher Logarithmus
log10 Dekadischer Logarithmus
log1p Berechnet log(1 + Argument) auch für sehr kleine Werte
log2 Logarithmus zur Basis 2
nearbyint Rundet zur nächsten Ganzzahl, bei .5 Richtung positiv unendlich
round Rundet zur nächsten Ganzzahl, bei .5 weg von 0
sin Sinus
sinh Sinus Hyperbolikus
sqrt Quadratwurzel
tan Tangens
tanh Tangens Hyperbolikus
tgamma Integral von 0 bis unendlich über t ^ (x - 1) e ^ -t dt
trunc Rundet zur nächsten Ganzzahl, immer in Richtung 0

Alle zweistelligen Funktionen:
atan2 Arkustangens von 2.Argument / 1.Argument
dim Positive Differenz beider Argumente: max(x - y, 0)
hypot Länge der Hypotenuse eines rechtwinkligen Dreiecks aus den Längen der beiden Katheten
jn Bessel-Funktion erster Art des 2. Arguments der Ordnung des 1. Arguments
max Größere der beiden Zahlen
min Kleinere der beiden Zahlen
mod Rest der Division 1. durch 2. Argument: x - n * y, wobei n = x / y gerundet zur nächsten Ganzzahl in Richtung 0
pow Potenz des ersten zur Basis des zweiten Arguments
remainder Rest der Division 1. durch 2. Argument: x - n * y, wobei n = x / y gerundet zur nächsten Ganzzahl
yn Bessel-Funktion zweiter Art des 2. Arguments der Ordnung des 1. Arguments

Hilfreich bei der Fehlersuche ist das Aktivieren von "Show detailed Log" im Hauptfenster:
Bei jedem Klick auf "OK" oder "Apply" nach Eingabe eines Ausdrucks im Konfigurationsdialog wird das Ergebnis der Parsers als Debugmeldung im Log des Hauptfensters ausgegeben.

Ausdrücke können außerdem durch Ausführen von "exprparser" oder "exprparser.exe" ohne PleoCommand mit beliebigen Eingabewerten getestet werden.
Das Programm befindet sich in "bin/ExpressionParser" oder auch "dist" nach Ausführen von "ant dist".
Die Syntax lautet "exprparser <Expression> [<Channel 1> <Channel 2> ...]".

Beispiele:
#0 <= 100 || #0 >= 200 1 für Zahlen außerhalb des Bereichs [100 .. 200], 0 sonst
cos(5 * log(#0 + sin(-#1))) Mehrfach verschachtelte Funktionen
rand() + !trunc(rand() * 5) * 100 Zufällige Zahlen zwischen 0 und 1, etwa jede 5. um 100 erhöht
