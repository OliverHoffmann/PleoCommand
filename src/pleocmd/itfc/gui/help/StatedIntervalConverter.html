Stated Interval Converter

Sendet ein oder (alternierend) zwei Kommandos mit einer auf Basis der eingehenden Daten berechneten Frequenz. Es werden Single-Float Datenpakete erwartet, deren Kehrwert die zu verwendende Frequenz ergeben: Ergibt oder übersteigt die Summe der Kehrwerte der gesammelten Datenpakete eins, dann wird ein Kommando ausgegeben und die Summierung beginnt von vorne.
Beispiele( "C" steht für ein gesendetes Kommando, "-" bedeutet, dass keine Ausgabe erfolgt:
[1, 1, 1, 1] erzeugt [C1, C2, C1, C2]
[3, 3, 3, 3, 3, 3] erzeugt [-, -, C1, -, -, C2]
[2, 2, 2, 2, 1, 1] erzeugt [-, C1, -, C2, C1, C2]
[1, 3, 3, 4, 4, 1000] erzeugt [C1, -, -, -, -, C2, -]

Ist das zweite Kommando nicht definiert (leerer String), so wird immer C1 verwendet.

Eine Übersicht über alle möglichen Befehle bietet das Dokument "Ugobe Pleo Monitor", welches Bestandteil des PleoDevelopmentKits ist (dieses befindet sich im Ordner "ext/dev" von PleoCommand).
Durch Eingabe von "PMC|help" kann auch direkt eine Liste aller möglichen Kommandos abgefragt werden.
